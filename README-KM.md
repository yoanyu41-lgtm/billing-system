## 🎨 ពណ៌ Brand

### ពណ៌ដែលគួរប្រើ (Recommended Colors)

| ពណ៌ | Hex Code | ប្រើសម្រាប់ | ឧទាហរណ៍ |
|------|----------|-------------|----------|
| 🟢 **បៃតង (Green)** | `#059669` | បង់រួច, ជោគជ័យ, ផ្ទៀងផ្ទាត់ | `<span class="badge-paid">បង់រួច</span>` |
| 🔴 **ក្រហម (Red)** | `#DC2626` | លើសកំណត់, មិនទាន់បង់, កំហុស | `<span class="badge-overdue">លើសកំណត់</span>` |
| 🟡 **លឿង (Yellow)** | `#D97706` | កំពុងរង់ចាំ, ការព្រមាន | `<span class="badge-pending">កំពុងរង់ចាំ</span>` |
| 🔵 **ខៀវ (Blue)** | `#2563EB` | ព័ត៌មាន, សកម្មភាពចម្បង | `<button class="btn-brand-blue">មើលព័ត៌មាន</button>` |
| ⚫ **ប្រផេះ (Gray)** | `#64748B` | មិនសកម្ម, បិទប្រើប្រាស់ | `<span class="badge-inactive">មិនសកម្ម</span>` |

### ស្ថានភាពការបង់ប្រាក់ (Payment Status)

```html
<!-- បង់រួចហើយ -->
<span class="pill pill-paid">បង់រួច</span>

<!-- លើសកំណត់ -->
<span class="pill pill-overdue">លើសកំណត់</span>

<!-- កំពុងរង់ចាំ -->
<span class="pill pill-pending">កំពុងរង់ចាំ</span>

<!-- កំពុងដំណើរការ -->
<span class="pill pill-ongoing">កំពុងដំណើរការ</span>
```

### ពណ៌ដែលអត់គួរប្រើ (Colors to Avoid)

❌ **ពណ៌ភ្លឺខ្លាំង (Neon Colors)**
- មូលហេតុ: ឈឺភ្នែក, រំខាន user
- ឧទាហរណ៍: Neon green, Neon pink, Neon yellow

❌ **ផ្ទៃខ្មៅសុទ្ធ (Pure Black)**
- មូលហេតុ: UI មើលធ្ងន់, មិន friendly
- លើកលែង: Dark Mode ដែលរចនាត្រឹមត្រូវ

❌ **ពណ៌ស្វាយ (Purple)**
- មូលហេតុ: មិនសម professional សម្រាប់ payment system

❌ **ពណ៌កខ្វក់ (Brown/Muddy)**
- មូលហេតុ: មិនស្អាតសម្រាប់ finance system

❌ **ពណ៌ច្រើនគ្មានប្រព័ន្ធ**
- មូលហេតុ: ច្រឡំ status (បង់រួច/មិនទាន់បង់ មើលដូចគ្នា)

## 🔤 ពុម្ពអក្សរ (Fonts)

### ការកំណត់ពុម្ពអក្សរ (Font Configuration)

| ភាសា | ពុម្ពអក្សរ | ប្រភេទ |
|------|-----------|--------|
| **អង់គ្លេស (English)** | Poppins | Sans-serif |
| **ខ្មែរ (Khmer)** | Khmer OS Battambang (Battambang) | Khmer Unicode |

### ការដំឡើងពុម្ពអក្សរ (Font Setup)
ប្រព័ន្ធនេះប្រើប្រាស់ Google Fonts សម្រាប់ទាញយកពុម្ពអក្សរដោយស្វ័យប្រវត្តិតាមរយៈអ៊ីនធឺណិត (Webfonts) ដូច្នេះមិនទាមទារឱ្យមានការទាញយក ឬដំឡើង Font នៅក្នុងម៉ាស៊ីនផ្ទាល់ខ្លួនឡើយ។ ប្រព័ន្ធនឹងដំណើរការដោយស្វ័យប្រវត្តិតែម្ដង។

## 💻 ការប្រើប្រាស់ (Usage)

ក្នុង layout blade file របស់អ្នក:

```blade

    <!-- ពុម្ពអក្សរ (Fonts) -->
    <link rel="stylesheet" href="{{ asset('css/fonts.css') }}">
    
    <!-- ពណ៌ Brand (Brand Colors) -->
    <link rel="stylesheet" href="{{ asset('css/brand-colors.css') }}">
</head>
```

### 2. ប្រើអត្ថបទខ្មែរ (Using Khmer Text)

```html
<!-- អត្ថបទខ្មែរសុទ្ធ -->
<p lang="km">សូមស្វាគមន៍មកកាន់ប្រព័ន្ធគ្រប់គ្រងវិក័យប័ត្រ</p>

<!-- អត្ថបទអង់គ្លេសសុទ្ធ -->

<!-- អត្ថបទចម្រុះ (Mixed) -->

    Welcome និងសូមស្វាគមន៍
</div>
```

# 📅 ការពន្យល់អំពី Installment Plans (ផែនការបង់រំលោះ)

## 🎯 Installment Plans គឺជាអ្វី?

**Installment Plans** (ផែនការបង់រំលោះ) គឺជាប្រព័ន្ធដែលអនុញ្ញាតឲ្យអតិថិជនទិញផលិតផលដោយមិនចាំបាច់បង់ប្រាក់ពេញលេញតែម្តង។ ពួកគេអាចបង់ជាដំណាក់កាលតូចៗ (រាល់ខែ) រហូតដល់បញ្ចប់។

### ឧទាហរណ៍ជីវិតប្រចាំថ្ងៃ:
```
អ្នកចង់ទិញកុំព្យូទ័រតម្លៃ $1,200
- មិនចាំបាច់បង់ $1,200 ទាំងអស់ពេលនេះ
- អាចបង់ដំណាក់កាល $100/ខែ រយៈពេល 12 ខែ
```

---

## 📋 សមាសភាគសំខាន់ (Key Components)

### 1. តម្លៃសរុប (Total Price)
ចំនួនទឹកប្រាក់ពេញលេញនៃផលិតផល
```
ឧទាហរណ៍: Laptop Dell = $800
```

### 2. ប្រាក់កក់ (Down Payment)
ចំនួនទឹកប្រាក់ដែលបង់នៅពេលចាប់ផ្តើម
```
ឧទាហរណ៍: បង់មុន $200
នៅសល់: $800 - $200 = $600
```

### 3. អត្រាការប្រាក់ (Interest Rate)
ភាគរយនៃការប្រាក់ក្នុងមួយឆ្នាំ
```
ឧទាហរណ៍: អត្រាការប្រាក់ = 10% ក្នុងមួយឆ្នាំ
```

### 4. រយៈពេល (Duration)
ចំនួនខែដែលត្រូវបង់រំលោះ
```
ឧទាហរណ៍: បង់រំលោះ 6 ខែ
```

### 5. ការបង់ប្រាក់ប្រចាំខែ (Monthly Payment)
ចំនួនទឹកប្រាក់ដែលត្រូវបង់រាល់ខែ
```
ឧទាហរណ៍: $100/ខែ × 6 ខែ = $600
```

---

## 🧮 រូបមន្តគណនា (Calculation Formula)

### 1. គណនាប្រាក់ដើម (Principal)
```
ប្រាក់ដើម = តម្លៃសរុប - ប្រាក់កក់
Principal = Total Price - Down Payment

ឧទាហរណ៍:
Principal = $800 - $200 = $600
```

### 2. គណនាការប្រាក់ប្រចាំខែ (Monthly Interest)
```
ការប្រាក់/ខែ = (ប្រាក់ដើម × អត្រាការប្រាក់% / 100) / 12
Interest/Month = (Principal × Rate / 100) / 12

ឧទាហរណ៍:
Interest/Month = ($600 × 10 / 100) / 12
Interest/Month = $60 / 12 = $5/ខែ
```

### 3. គណនាការបង់ប្រាក់ប្រចាំខែ (Monthly Payment)
```
ការបង់/ខែ = (ប្រាក់ដើម / ចំនួនខែ) + ការប្រាក់/ខែ
Payment/Month = (Principal / Months) + Interest/Month

ឧទាហរណ៍:
Payment/Month = ($600 / 6) + $5
Payment/Month = $100 + $5 = $105/ខែ
```

## 👥 ការគ្រប់គ្រងអតិថិជន និងការវាយតម្លៃឥណទាន (Customer & Credit Assessment)

### ៦.១ ព័ត៌មានអតិថិជន (Customer Profile)
ដើម្បីចាប់ផ្តើមគម្រោងបង់រំលោះ ឬលក់ទំនិញ ព័ត៌មានអតិថិជនត្រូវតែកត់ត្រាទុកក្នុងប្រព័ន្ធ៖
*   **ព័ត៌មានផ្ទាល់ខ្លួន៖** ឈ្មោះ ភេទ ថ្ងៃខែឆ្នាំកំណើត លេខទូរស័ព្ទ អាសយដ្ឋាន និងមុខរបរ។
*   **ឯកសារយោង៖** លេខអត្តសញ្ញាណប័ណ្ណ (National ID) រូបថតអតិថិជន និងរូបថតអត្តសញ្ញាណប័ណ្ណ។
*   **ចំណូលប្រចាំខែ៖** ប្រើសម្រាប់វាយតម្លៃសមត្ថភាពទូទាត់។

### ៦.២ ការវាយតម្លៃឥណទាន (Credit Check)
មុនពេលអនុម័តគម្រោងបង់រំលោះ បុគ្គលិកត្រូវធ្វើការវាយតម្លៃឥណទានអតិថិជន៖
*   **ការផ្ទៀងផ្ទាត់៖** ផ្ទៀងផ្ទាត់កន្លែងធ្វើការ (Employment) និងប្រភពចំណូល (Income)។
*   **សមាមាត្របំណុល (Debt Ratio)៖** គណនាផលធៀបបំណុល និងចំណូល ដើម្បីចៀសវាងការជំពាក់លើសសមត្ថភាព។
*   **កម្រិតហានិភ័យ (Risk Level)៖**
    *   `Low Risk` (ហានិភ័យទាប)៖ អាចអនុម័តគម្រោងបង់រំលោះលក្ខខណ្ឌធម្មតា។
    *   `Medium Risk` (ហានិភ័យមធ្យម)៖ ទាមទារឱ្យមាន **អ្នកធានា (Guarantor)**។
    *   `High Risk` (ហានិភ័យខ្ពស់)៖ ទាមទារប្រាក់កក់ខ្ពស់ជាងមុន ឬត្រូវបដិសេធ។

### ៦.៣ ព័ត៌មានអ្នកធានា (Guarantor)
សម្រាប់ករណីហានិភ័យមធ្យម ឬខ្ពស់ អតិថិជនត្រូវមានអ្នកធានាដែលត្រូវផ្តល់ព័ត៌មានលម្អិតដូចគ្នា (ឈ្មោះ លេខទូរស័ព្ទ មុខរបរ ចំណូលប្រចាំខែ និងទំនាក់ទំនងជាមួយអតិថិជន)។

## 📦 ការគ្រប់គ្រងផលិតផល និងស្តុក (Product & Inventory)

### ៧.១ ព័ត៌មានបច្ចេកទេសផលិតផល (Computer Specifications)
ដោយសារប្រព័ន្ធនេះរចនាឡើងសម្រាប់ហាងកុំព្យូទ័រ ផលិតផលនីមួយៗអាចបញ្ចូលព័ត៌មានលម្អិតដូចជា៖
*   **លក្ខណៈបច្ចេកទេស៖** CPU, RAM, Storage, GPU, Displays, Battery
*   **ការធានា និងស្ថានភាព៖** រយៈពេលធានា (Warranty), ស្ថានភាពទំនិញ (ថ្មី/ជជុះ/កែច្នៃឡើងវិញ)

### ៧.២ លំហូរចូលស្តុក (Stock In / Purchases)
*   **Supplier Management៖** គ្រប់គ្រងព័ត៌មានអ្នកផ្គត់ផ្គង់ (ឈ្មោះ លេខទូរស័ព្ទ អាសយដ្ឋាន)។
*   **Purchase Orders៖** បង្កើតបញ្ជាទិញចូលស្តុក កត់ត្រាតម្លៃដើម (Cost Price) និងចំនួនទំនិញ។
*   **Stock In Update៖** នៅពេលទំនិញដឹកមកដល់ បុគ្គលិកប្តូរស្ថានភាពទៅជា `Received` នោះចំនួនស្តុកក្នុងប្រព័ន្ធនឹងកើនឡើងដោយស្វ័យប្រវត្តិ។

### ៧.៣ លំហូរចេញស្តុក (Stock Out / Sales)
*   នៅពេលមានការលក់ទំនិញ (លក់ផ្ទាល់ ឬលក់រំលោះ) ចំនួនស្តុកនឹងត្រូវដកចេញដោយស្វ័យប្រវត្តិតាមចំនួនដែលបានលក់។
*   **Stock Movement Logs៖** រាល់ការប្រែប្រួលស្តុក (ចូល/ចេញ/តម្រូវស្តុក) នឹងត្រូវកត់ត្រាទុកក្នុងប្រព័ន្ធដើម្បីងាយស្រួលស៊ើបអង្កេត។
*   **Low Stock Alert៖** ប្រព័ន្ធនឹងបង្ហាញសារព្រមាននៅពេលទំនិញណាមួយធ្លាក់ចុះដល់កម្រិតកំណត់ (Low Stock Threshold)។

## 🛍️ ការលក់ផ្ទាល់ (Direct Sales / POS)

### ៨.១ ដំណើរការលក់ផ្ទាល់
សម្រាប់អតិថិជនដែលទិញទំនិញបង់លុយផ្តាច់ (មិនបង់រំលោះ)៖
1.  ទៅកាន់ menu **Stock Out** ឬ **Direct Sales**។
2.  ជ្រើសរើសអតិថិជន (អតិថិជនទូទៅ/Walk-in ឬអតិថិជនសមាជិក)។
3.  ជ្រើសរើសទំនិញ និងបញ្ជាក់ចំនួន។
4.  ប្រព័ន្ធនឹងបង្ហាញ Subtotal, Discount និងពន្ធដារ។
5.  ជ្រើសរើសវិធីទូទាត់ប្រាក់ (សាច់ប្រាក់, ABA, Wing...) រួចចុច **Complete Sale**។

### ៨.២ ការគណនាពន្ធអាករ (VAT/Tax Calculation)
ប្រព័ន្ធគាំទ្រការកំណត់ពន្ធពីរប្រភេទ៖
*   **Exclusive Tax (មិនរួមពន្ធ)៖** ពន្ធ (ឧទាហរណ៍ ១០%) នឹងត្រូវបូកបន្ថែមពីលើតម្លៃទំនិញ។
*   **Inclusive Tax (រួមពន្ធរួចហើយ)៖** តម្លៃទំនិញរួមបញ្ចូលពន្ធរួចជាស្រេច ហើយប្រព័ន្ធនឹងបំបែកតម្លៃដើម និងពន្ធដាច់ដោយឡែកលើវិក្កយបត្រ។

### ៨.៣ ការបោះពុម្ពវិក្កយបត្រ (Invoice Printing)
ក្រោយពេលលក់រួច ប្រព័ន្ធនឹងបង្កើតវិក្កយបត្រស្វ័យប្រវត្តិ ដែលអាច៖
*   មើលព័ត៌មានលម្អិតនៃការលក់តាម Browser។
*   ទាញយកជាឯកសារ PDF ឬបោះពុម្ពចេញតាមម៉ាស៊ីនព្រីន (Print Receipt)។

## ✈️ មជ្ឈមណ្ឌល Telegram និងការជូនដំណឹង (Telegram Bot & Notifications)

### ៩.១ មុខងារស្វ័យប្រវត្តិនៃ Telegram Bot
ប្រព័ន្ធប្រើប្រាស់ Telegram Bot API ដើម្បី៖
1.  **ផ្ញើសារជូនដំណឹងការបង់ប្រាក់៖** ជូនដំណឹងដល់អតិថិជន ៣ ថ្ងៃមុនកាលកំណត់, នៅថ្ងៃកំណត់ និង ១ ថ្ងៃក្រោយកាលកំណត់ (Overdue)។
2.  **ផ្ញើបង្កាន់ដៃបង់ប្រាក់៖** ផ្ញើសារអបអរសាទរ និងបញ្ជាក់ការទទួលប្រាក់ភ្លាមៗនៅពេល Admin យល់ព្រមលើការបង់ប្រាក់។
3.  **ស្វែងរកសមតុល្យ៖** អតិថិជនអាចវាយពាក្យបញ្ជា `/balance` លើ Bot ដើម្បីឆែកមើលទឹកប្រាក់ដែលនៅជំពាក់ និងកាលបរិច្ឆេទត្រូវបង់បន្ទាប់។

### ៩.២ ការកំណត់ប្រព័ន្ធ Telegram Webhook
*   ដើម្បីឱ្យ Bot ដំណើរការបាន ត្រូវបញ្ចូល **Telegram Bot Token** នៅក្នុងទំព័រ Settings។
*   ចុចលើប៊ូតុង **Set Webhook** ក្នុងទំព័រ Telegram Center ដើម្បីភ្ជាប់ប្រព័ន្ធ Laravel ទៅកាន់ Telegram Servers។

## ⚙️ ការកំណត់ប្រព័ន្ធ និងការចម្លងទុកទិន្នន័យ (Settings & Backup)

### ១០.១ គណនីបុគ្គលិក និងតួនាទី (Role-Based Access Control)
*   **Admin (អ្នកគ្រប់គ្រងជាន់ខ្ពស់)៖** គ្រប់គ្រងមុខងារទាំងអស់ រួមទាំងការកំណត់ប្រព័ន្ធ, ការចម្លងទុកទិន្នន័យ, គ្រប់គ្រងគណនីបុគ្គលិក និងមើលរបាយការណ៍ហិរញ្ញវត្ថុ។
*   **Manager (ប្រធានផ្នែក)៖** អាចមើលរបាយការណ៍លក់, គ្រប់គ្រងស្តុក, និងអនុម័តការទូទាត់ប្រាក់ (Approve Payments)។
*   **Staff (បុគ្គលិកទូទៅ)៖** អាចកត់ត្រាការលក់, បង្កើតគម្រោងបង់រំលោះ, និងបញ្ចូលការបង់ប្រាក់ (តែត្រូវរង់ចាំការអនុម័តពី Manager/Admin)។

### ១០.២ ការចម្លងទុកទិន្នន័យ (Backup & Restore)
*   Admin អាចទាញយកឯកសារ Backup (.sql) ឬចុចលើប៊ូតុង **Restore** ដើម្បីសង្គ្រោះទិន្នន័យត្រឡប់មកវិញក្នុងករណីមានបញ្ហា។

### ១០.៣ ការកំណត់ក្រុមហ៊ុន (Company Settings)
*   អនុញ្ញាតឱ្យផ្លាស់ប្តូរឈ្មោះហាង (ជាភាសាខ្មែរ និងអង់គ្លេស) អាសយដ្ឋាន លេខទំនាក់ទំនង និង Logo ហាង ដែលទិន្នន័យទាំងនេះនឹងត្រូវបង្ហាញនៅលើកិច្ចសន្យា និងវិក្កយបត្រដោយស្វ័យប្រវត្តិ។
